# 시스템 오류 vs AI 오류 비교

## context/error.md (AI 오류 로그)
- **목적**: AI의 실수와 오류를 기록하고 학습
- **대상**: AI가 범한 오류 (정보 제공 실수, 판단 오류 등)
- **특징**:
  - 간결한 형식: [E0X] 코드로 분류
  - 상황-실수-개선의 3단계 구조
  - AI의 행동 지침 개선 목적
  - 개인적인 학습 기록 성격

## context/errors/ (시스템 오류 관리)
- **목적**: 시스템/API의 기술적 오류를 관리하고 재발 방지
- **대상**: 외부 시스템의 버그, API 오류, 인프라 문제
- **특징**:
  - 상세한 기술 문서 형식
  - 원인-해결-예방의 구조화된 접근
  - 팀 단위의 협업을 위한 프로세스
  - 코드 예시, 모니터링, 장기 계획 포함

---

## 주요 차이점

| 항목 | AI 오류 로그 (error.md) | 시스템 오류 관리 (errors/) |
|------|------------------------|--------------------------|
| **범위** | 개인 AI의 학습 | 팀/시스템 기술 관리 |
| **형식** | 간결한 텍스트 | 구조화된 문서 |
| **심각도** | 정보 정확성 | 시스템 안정성 |
| **해결 방법** | 개인 개선 계획 | 기술 솔루션 구현 |
| **공유 대상** | AI 자신 | 개발팀 전체 |
| **수명 주기** | 즉각적 학습 | 장기적 유지보수 |

## 올바른 사용 예시

### AI 오류 (error.md에 기록)
```
[E01] 2026-03-09
상황: 3월 9일 아침 브리핑 중 날짜/요일 파악
실수: 3월 9일을 일요일이라고 계속 착각함 (실제는 월요일)
개선: 날짜/요일 계산 시 사용자가 명시한 정보를 절대 의심하지 않기
```

### 시스템 오류 (errors/에 기록)
- 캘린더 API가 잘못된 날짜의 이벤트를 반환
- 네트워크 연결 실패로 데이터 누락
- 서버 응답 지연으로 타임아웃 발생

## 연동 방안

두 시스템은 상호 보완적이어야 함:

1. **AI 오류가 시스템 오류로 발전하는 경우**
   - AI가 API 오류를 잘못 해석하면 error.md에 기록
   - 원인이 시스템 문제임이 밝혀지면 errors/로 상세 기록 이관

2. **시스템 오류에서 AI 학습**
   - errors/에 기록된 패턴을 error.md에 간단히 요약
   - AI가 비슷한 상황에서 더 나은 판단을 하도록 학습

3. **정기적인 동기화**
   - 월간 리뷰에서 두 오류 로그를 함께 검토
   - 반복 패턴을 AI_identity.md에 반영
