# 오류 관리 시스템

이 폴더는 시스템에서 발생한 오류들을 체계적으로 관리하고 재발 방지를 위한 공간입니다.

## 폴더 구조

```
errors/
├── README.md                 # 이 파일 (오류 관리 가이드)
├── calendar_api_error_report.md  # 캘린더 API 오류 레포트
├── template_error_report.md      # 오류 레포트 템플릿
└── resolved/                     # 해결된 오류들
```

## 오류 레포트 작성 가이드

### 1. 파일명 규칙
- 형식: `YYYY-MM-DD_오류종류_간단설명.md`
- 예: `2026-03-09_calendar_api_recurring_events.md`

### 2. 필수 포함 항목
```markdown
# 오류 제목

## 발생 일시
- 시간: YYYY-MM-DD HH:MM
- 담당자: [담당자 이름]

## 오류 현상
- 구체적인 증상 기술
- 재현 단계
- 스크린샷/로그 첨부

## 원인 분석
- 근본 원인
- 영향 범위
- 관련 시스템/모듈

## 해결 방안
1. 긴급조치 (즉시)
2. 단기 조치 (1-2주)
3. 중기 조치 (1개월)
4. 장기 조치 (3개월 이상)

## 재발 방지 대책
- 프로세스 개선
- 코드 레벨 대책
- 모니터링 방안
- 교육 필요 사항
```

## 오류 처리 프로세스

### 1. 오류 발생 시
1. 즉시 템플릿으로 레포트 작성
2. 이슈 트래커 등록 (JIRA/GitHub Issues)
3. 관련 팀원에게 알림

### 2. 해결 진행 중
1. 정기적인 진행 상황 업데이트
2. 테스트 케이스 작성
3. 롤백 계획 수립

### 3. 해결 완료 시
1. 해결 내용 상세 기록
2. `resolved/` 폴더로 이동
3. 후속 조치 일정 설정
4. 지식베이스에 등록
