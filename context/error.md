# 에러 로그

> Hai가 실수하거나 잘못된 정보를 제공했을 때 기록한다.
> 반복되는 패턴은 `context/ai_identity.md` 행동 지침에 반영한다.

---

## 에러 유형

| 코드 | 유형 | 설명 |
|------|------|------|
| E01 | 시간/날짜 | 요일 착각, 날짜 계산 오류 |
| E02 | 데이터 필터링 | 조건 누락, 잘못된 필터 |
| E03 | API/MCP | 연결 실패, 파싱 오류 |
| E04 | 정보 손실 | 삭제, 덮어쓰기 |
| E05 | 정보 확인 누락 | 지식베이스 미확인 후 답변 |
| E06 | 프로세스 위반 | 원칙/스킬 미준수 |
| E07 | 도구 호환성 | 플랫폼별 차이 미확인 |

---

## 기록 형식

```
[E0X] YYYY-MM-DD
상황: 어떤 맥락에서 발생했는가
실수: 무엇을 잘못했는가
개선: 다음엔 어떻게 할 것인가
```

---

## 로그

[E01] 2026-03-09
상황: 3월 9일 아침 브리핑 중 날짜/요일 파악
실수: 3월 9일을 일요일이라고 계속 착각함 (실제는 월요일)
개선:
- 날짜/요일 계산 시 사용자가 명시한 정보를 절대 의심하지 않기
- "오늘이 3월 9일 월요일"이라는 기준점만 확보되면 다른 날은 상대적 계산 가능
- 일정 브리핑 시 현재 날짜를 항상 명확히 인지하고 시작할 것
- MCP 캘린더 조회 결과와 상관없이 사용자의 날짜 정보를 최우선으로 신뢰할 것

[해결책] 기준점 확립:
- 오늘: 2026년 3월 9일 월요일
- 따라서 어제: 3월 8일 일요일
- 내일: 3월 10일 화요일
