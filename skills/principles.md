# 코딩 원칙

## 1. 관심사의 분리 / SRP (Single Responsibility Principle)
모든 함수, 컴포넌트, 모듈은 단 하나의 책임만 가진다.
하나의 이유로만 변경되어야 한다.

## 2. 프레임워크 독립적인 핵심 로직
핵심 비즈니스 로직은 프레임워크에 의존하지 않는다.
프레임워크 교체 시 핵심 로직을 건드리지 않아도 되는 구조를 지향한다.
- UI 레이어 (Nuxt/Next) ← 비즈니스 로직 ← 데이터/인프라 레이어 순으로 의존

## 3. 리팩토링이 용이한 폴더 및 파일 구조
- 일관된 네이밍 컨벤션으로 일괄 찾아바꾸기 가능한 구조
- 예측 가능한 파일 위치 (어디에 뭐가 있는지 규칙으로 유추 가능)
- 패턴이 통일되어 있어야 함

## 4. 단방향 의존성 (No Upward Dependency)
레이어 구조에서 의존 방향은 항상 단방향이다.
하위 레이어가 상위 레이어를 참조해서는 안 된다.
순환 의존(Circular Dependency)은 절대 금지.

```
UI 컴포넌트
    ↓
비즈니스 로직 / 서비스
    ↓
데이터 레이어 / 인프라
```

## 5. 임의 결정 금지
구조, 패턴, 네이밍 등 설계 결정은 반드시 사용자에게 먼저 물어본다.
제안 시 왜 그 방향인지 근거를 함께 제시한다.
